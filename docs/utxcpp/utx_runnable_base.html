utx::runnable_base

utx::runnable_base is an abstract c++ base class, used to implement c++ object-oriented thread runnable.

.start_thread

.start_thread is method of utx::runnable_base, should only be called by .start, .start_thread is used to create native_thread_type, native_thread_type is commonly std::thread or std::jthread.

.start_thread is final, can not be overrided.

.start_thread should not be called by user code.

.start

.start is the entry point of launching a thread, to call .start_thread,

.start is final, can not be overrided.

.start should be called by user code.

.run_thread

.run_thread is method of utx::runnable_base, to used as the callback function of native-thread-type std::thread or std::jthread. .run_thread will call .run method.

.run_thread is final, can not be overrided.

.run_thread should be not called by user code.

.run

.run method is called by method .run_thread, .run is the customized sub-routine of the thread callback.

.run is pure virtual, should be overrrided by final thread class.

.run should be not called by user code.

.join

.join is the method of utx::runnable_base, used to wait and join current thread to parent thread, .join will check .joinable first, then call .join from native-type (std::thread or std::jthread).

.join is final, can not be overrrided.

.join can be called by both user code, and utxcpp thread api.

.get_id

.get_id is method of utx::runnable_base to get the current running thread id. .get_id will return thread-id dynamically and should be, the thread does not statically store the thread id.

.get_id is final, can not be overrided.

.get_id can be called by both user code and utxcpp thread api.

.enable_tcrash

.enable_tcrash is method of utx::runnable_base to enable or disable __enable_tcrash, when it is enabled, exception caught will be thrown again without further catch, otherwise exception will be caught by catch and handled.

.enable_tcrash is final, can not be overrided.

.enable_tcrash can be called by both user code and utxcpp thread api.
