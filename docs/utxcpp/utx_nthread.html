utx::nthread

utx::nthread is derived from utx::nthread_base, which uses std::thread as its native thread. utx::nthread has different interface with other thread api, can have different features.

utx::nthread_base is the object-oriented thread design of c++ utxcpp, unlike std::thread. However, utx::nthread_base uses std::thread as its native thread type.

Public callable Methods

  • thread.start()
  • thread.join()
  • thread.add_task(task, auto_start__?)
  • thread.get_id()
  • thread.enable_tcrash(true/false)

Public Methods

These methods can be called from user code.

thread.start()

Start the thread. This method should be called only if the thread is valid and not started.

.start is the entry point of launching a thread, to call .start_thread,

.start is final, can not be overrided.

In utx::abstract_thread_base, .start is not and should not be called by constructor, in its derived class, .start can be called by constructor or user code.

thread.join()

Join (wait) the thread to parent thread.

.join is the method of utx::abstract_thread_base, used to wait and join current thread to parent thread.

.join is pure virtual, as different native threads have different join method, the derived class should override .join(). Such as std::thread and std::jthread use .join(), std::future uses .wait().

To implement .join() in its derived class, joinable checking is suggested to use before its wait, then call .join() from native-type (std::thread or std::jthread).

thread.add_task(task, auto_start__?)

Add a task to the thread. This method should be called if the thread has no task or the previous task is finished.

thread.get_id()

Return the thread id realtime. The thread object can not hold its id statically, every time calling .get_id(), the id is fetched realtime.

.get_id is method of utx::abstract_thread_base to get the current running thread id. .get_id will return thread-id dynamically and should be, the thread does not statically store the thread id.

.get_id is pure virtual, as different native threads (std::thread, std::jthread, std::future) have different get_id way.

.get_id can be called by both user code and utxcpp thread api.

thread.enable_tcrash(true/false)

By default, the exception will be handled and proccessed automatically by the api, but this method .enable_tcrash can change that.

.enable_tcrash is method of utx::abstract_thread_base to enable or disable __enable_tcrash, when it is enabled, exception caught will be thrown again without further catch, otherwise exception will be caught by catch and handled.

.enable_tcrash is final, can not be overrided.

.enable_tcrash can be called by both user code and utxcpp thread api.

Protected Methods

These methods should not be called from user code.

thread.start_thread()

.start_thread is method of utx::abstract_thread_base, should only be called by .start, .start_thread is used to create and launch native_thread_type thread, native_thread_type is commonly std::thread, std::jthread or std::future etc.

.start_thread is pure virtual, must be overrided.

.start_thread should not be called by user code.

.start_thread should be called by .start

thread.run()

.run is method of utx::abstract_thread_base, used as the callback function of native-thread-type std::thread or std::jthread. .run will call .run_task method.

.run is final, can not be overrrided.

.run should be not called by user code.

thread.run_task()

.run_task method is called by method .run, .run_task is the sub-routine of the thread callback.

.run_task is final, can not be overrided.

.run_task should not be called by user code.
