utx::string_literals

utx::string_literals::operator""_us

operator""_us is the string literal of utxcpp string. It forms one of utx::string, utx::u8string, utx::u16string, utx::u32string or utx::wstring from a literal string. 

inline utx::string operator""_us(const char * str, utx::u64 size);
inline utx::u8string operator""_us(const char8_t * str, utx::u64 size);
inline utx::u16string operator""_us(const char16_t * str, utx::u64 size);
inline utx::u32string operator""_us(const char32_t * str, utx::u64 size);
inline utx::wstring operator""_us(const wchar_t * str, utx::u64 size);

utx::string_literals are available for:

  • utx::string
  • utx::u8string
  • utx::u16string
  • utx::u32string
  • utx::wstring

"using namespace utx::string_literals;" also means "using namespace std::string_literals;"

c++ example 

#include <utxcpp/core.hpp>

using namespace utx::string_literals;

int main()
{
	auto str1 = "Hello, c++!"_us;
	auto str2 = "Hello, c++!"_us.cat(" ", 123, ' ', true, " ", 7, "kg");
	utx::print(str1, '\n', str2);
	/*
	Hello, c++!
	Hello, c++! 123 true 7kg
	*/
}
