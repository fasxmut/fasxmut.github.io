utx::thread_base

utx::thread_base

utx::thread_base is an abstract c++ base class, used to implement c++ object-oriented thread.

callable methods in user-code

  • .join()
  • .get_id()
  • .enable_crash()

.start

.start is the entry point of launching a thread, to call .start_thread,

.start is final, can not be overrided.

In utx::thread_base, .start is not called by constructor, in its derived class, .start can be called by constructor or user code.

.start_thread

.start_thread is method of utx::thread_base, should only be called by .start, .start_thread is used to create and launch native_thread_type thread, native_thread_type is commonly std::thread, std::jthread or std::future etc.

.start_thread is pure virtual, must be overrided.

.start_thread should not be called by user code.

.start_thread should be called by .start

.run

.run is method of utx::thread_base, used as the callback function of native-thread-type std::thread or std::jthread. .run will call .run_task method.

.run is final, can not be overrrided.

.run should be not called by user code.

.run_task

.run_task method is called by method .run, .run_task is the sub-routine of the thread callback.

.run_task is final, can not be overrided.

.run_task should not be called by user code.

difference of .run and .run_task, and why separate

.run is the sub-callback of the callback of native thread (std::thread, std::jthread, std::future<void>).

.run will do the task selection from the std::variant object;

.run_task will run the selected task of the utx::task_base based object.

.join

.join is the method of utx::thread_base, used to wait and join current thread to parent thread.

.join is pure virtual, as different native threads have different join method, the derived class should override .join(). Such as std::thread and std::jthread use .join(), std::future uses .wait().

To implement .join() in its derived class, joinable checking is suggested to use before its wait, then call .join() from native-type (std::thread or std::jthread).

.join can be called by both user code and utxcpp thread api.

.get_id

.get_id is method of utx::thread_base to get the current running thread id. .get_id will return thread-id dynamically and should be, the thread does not statically store the thread id.

.get_id is pure virtual, as different native threads (std::thread, std::jthread, std::future) have different get_id way.

.get_id can be called by both user code and utxcpp thread api.

.enable_tcrash

.enable_tcrash is method of utx::thread_base to enable or disable __enable_tcrash, when it is enabled, exception caught will be thrown again without further catch, otherwise exception will be caught by catch and handled.

.enable_tcrash is final, can not be overrided.

.enable_tcrash can be called by both user code and utxcpp thread api.
