utfs::copy_options objects

utfs::copy_options objects are defined by utfs::copy_options class, used to define copy options of utfs::copy and utfs::copy_all.

  • utfs::cpopt_none
  • utfs::cpopt_skip_existing
  • utfs::cpopt_overwrite_existing
  • utfs::cpopt_update_existing
  • utfs::cpopt_recursive
  • utfs::cpopt_copy_symlinks
  • utfs::cpopt_skip_symlinks
  • utfs::cpopt_directories_only
  • utfs::cpopt_create_symlinks
  • utfs::cpopt_create_hard_links
  • utfs::cpopt_best

none

  • utfs::cpopt_none

to file already exists

  • utfs::cpopt_skip_existing -------------- Keep the existing file, without reporting an error.
  • utfs::cpopt_overwrite_existing --------- Replace the existing file.
  • utfs::cpopt_update_existing ------------ Replace the existing file only if it is older than the file being copied.

to subdirectories

  • utfs::cpopt_recursive ------------------ Recursively copy subdirectories and their content.

to copy() symbolic links

  • utfs::cpopt_copy_symlinks -------------- Copy symlinks as symlinks, not as the files they point to.
  • utfs::cpopt_skip_symlinks -------------- Ignore symlinks.

to copying copy() symbolic links

  • utfs::cpopt_directories_only ----------- Copy the directory structure, but do not copy any non-directory files.
  • utfs::cpopt_create_symlinks ------------ Instead of creating copies of files, create symlinks pointing to the originals. Note: the source path must be an absolute path unless the destination path is in the current directory.
  • utfs::cpopt_create_hard_links ---------- Instead of creating copies of files, create hardlinks that resolve to the same files as the originals.

Best

  • utfs::cpopt_best ----------------------- The best options trying used by utfs::copy() and utfs::copy_file()
