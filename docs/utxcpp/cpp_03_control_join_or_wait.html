cpp 03. Control join (or wait)

Control thread join or wait of utxcpp thread, c++.

Method .join() of the thread will be automatically called on destructing, however, you can call it by hand to let the thread to join parent thread ahead of end.

Method .join() of utxcpp thread will call and check .joinable(), then call method .join() of native thread.

native thread includes std::thread, std::jthread, std::future<void> . 

{
	utx::thread thread{a_task};
} // thread.join() will be auto called on destructing.

{
	utx::thread thread{a_task};
	thread.join();	// call thread.join() explicitly ahead of end.
	// After .join(), status becomes not .joinable,
	// then destructor will not call native method .join().
}

Final cpp complete example 

#include <utxcpp/core.hpp>
#include <utxcpp/thread.hpp>

class simple_task: public utx::task
{
public:
	void run() override
	{
		std::cout << "a simple task\n";
	}
};

int main()
{
	utx::jthread thread{simple_task{}};
	thread.join();
}
