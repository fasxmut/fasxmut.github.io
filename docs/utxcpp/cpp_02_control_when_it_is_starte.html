cpp 02. Control when it is started

Control when the thread is started.

The thread will be automatically started and launched on constructing by default. However,

the thread will be automatically only if the following conditions are met:

  • You passed a task to the constructor of the thread - requires task is not empty
  • The flag auto_start__ passed to the thread constructor is true - it is true by default
  • No weird crash - crash is unknown

For example, the thread will be automatically started and launched like this: (suppose no weird crash here) 

utx::jthread thread{a_task};

How to create a thread without auto start?

Method-1: Pass false to auto_start__

Pass false as the second parameter of the thread constructor, to set auto_start__ to false: 

utx::thread thread{a_task, false};	// set auto_start__ to false

Then you must call method .start() to start the thread: 

thread.start();

Method-2: Create an empty task thread

Create a thread without task will be not started. 

utx::thread thread;	// the thread will be not started.

Call .start() then will start the empty thread in current api version, but I am not sure if I will disable empty task thread to be running in the future. 

thread.run();

Notice, you can not pass true or false to flag auto_start__ without a task passed: 

utx::thread thread{true};	// syntax error
utx::thread thread2{false};	// syntax error

Final cpp complete example 

#include <utxcpp/core.hpp>
#include <utxcpp/thread.hpp>

class simple_task: public utx::thread
{
public:
	void run() override
	{
		std::cout << "A simple task." << std::endl;
	}
};

int main()
{
	{
		simple_task s1;
		utx::thread t1{s1, false};	// t1 is not auto started.
		t1.start();	// start t1 now
	}
	{
		simple_task s1;
		utx::thread t1{s1, true};	// t1 is auto started.
	}
	{
		simple_task s1;
		utx::thread t1{s1};	// t1 is auto started by default.
	}
}
