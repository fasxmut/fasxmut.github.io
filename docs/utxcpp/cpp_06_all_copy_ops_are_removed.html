cpp 06. All copy OPs are removed

All copy OPs of the utxcpp c++ thread api are removed: they are marked as "=delete;", so you can not use methods such as .push_back and .emplace_back to push utxcpp thread to a STL container.

The copy OPs mentioned here include:

  • All copy constructors
  • All assignment operators

They are = delete;

std::vector's .push_back, .emplace_back is not available for the utxcpp threads

Because the copy OPs must be called when the std::vector and other stl containers' .push_back, .emplace_back etc. are called. But utxcpp threads' copy OPs are removed. 

utx::thread thread{a_task};
std::vector<utx::thread> v1;
v1.push_back(thread);	// error
v1.emplace_back(a_task);	// error

How to store utxcpp threads to an STL container

method 1

Pre-allocate the vector on constructing. 

std::vector<utx::thread> vector(10);
