kjcpp::sqrt

kjcpp::sqrt

kjcpp::sqrt calculates the square root of a real value. The parameter passed to kjcpp::sqrt can be any real type (kjcpp::real_meric), but the returned result is always floating type. If the parameter is negative, it returns NaN value (kjcpp::nan_v result).

Calling Sig : 

constexpr auto result = kjcpp::sqrt(value);

constexpr auto result = kjcpp::sqrt.fn<return_type>(value);

kjcpp::sqrt(value) :

  • It always returns a float value.
  • The returned float type is one of kjcpp::f32, kjcpp::f64, kjcpp::f128, kjcpp::fmax, kjcpp::fx32, kjcpp::fx64, kjcpp::fx128, kjcpp::fxmax.
  • If value is negative, it returns kjcpp::nan_v<return_float_type> .
  • If you want to check if the return value is nan, you'd better use kjcpp::is_nan(value), as the return_float_type is uncertainty.
  • constexpr support.

kjcpp::sqrt.fn<return_type>(value) :

  • The result will be converted to return_type .

Header 

#include <kjcpp/math.hpp>

c++ example 

#include <kjcpp/core.hpp>
#include <kjcpp/math.hpp>

int main()
{
	constexpr auto a = kjcpp::sqrt(3);
	constexpr auto b = kjcpp::sqrt(3.4);
	constexpr auto c = kjcpp::sqrt(kjcpp::ix32{2});
	constexpr auto d = kjcpp::sqrt(kjcpp::fx32{2.8});
	constexpr auto e = kjcpp::sqrt(-7);

	kjcpp::print(a,b,c,d,e);
/*output:
1.732051 1.843909 1.414214 1.673320 nan
*/

	static_assert(kjcpp::is_nan(e));
}
