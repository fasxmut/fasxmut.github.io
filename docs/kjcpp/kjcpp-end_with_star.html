kjcpp::end_with_star

kjcpp::end_with_star concept

kjcpp::end_with_star is a concept used to match a type which has a pointer star symbol at the end.

As some type might be a pointer but does not have a pointer star symbol, kjcpp::end_with_star is used to distinguish them, kjcpp::end_with_star concept guarantees a star symbol must exist at the end.

The following types satisfy concept kjcpp::end_with_star : 

const char *
char *
int *
any_type any_qualifiers *

The following types do not satisfy concept kjcpp::end_with_star : 

const char * const
char * const
int * const
chat (*)[16]

To match a type which has a star symbol before cv- qualifiers, you can use kjcpp::remove_cvref_t on it, then use concept kjcpp::end_with_star .

kjcpp::end_with_star was designed for kjcpp::str_size, but it can be used for general purpose.

c++ example 

#include <kjcpp/core.hpp>

template <kjcpp::end_with_star pointer_type>
class fizz_box
{
};

int main()
{
	fizz_box<const char *> f1;	// OK
	// fizz_box<char (*) [16]> f2;	// error
}
