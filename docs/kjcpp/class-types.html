kjcpp::class_type

kjcpp::class_type is used to define basic integral and floating point types, but it is class. So the data types defined by kjcpp::class_type have both the basic data type features and object-oriented features.

.value(), operator(), operator basic_type

These three member functions return __value by value.

.value_ref()

Member function .value_ref() returns __value by ref.

kjcpp::class_type does not overload .value() to support both by-value and by-ref, but uses .value_ref() to get by ref.

include

kjcpp/types.hpp : 

#include <kjcpp/types.hpp>

kjcpp/types.hpp is included by the whole system, so using kjcpp/core.hpp works too: 

#include <kjcpp/core.hpp>

examples

example: Using kjcpp::class_type, and constexpr support 

#include <kjcpp/core.hpp>

int main()
{
	constexpr kjcpp::ux64 a = 123;
	constexpr kjcpp::ix64 b = 321;
	constexpr kjcpp::fx64 c = 321.123;
	kjcpp::print(a, b, c);
}

output: 

123 321 321.123000

example: kjcpp::class_type value as template parameter 

#include <kjcpp/core.hpp>

template <auto __value>
class power
{
public:
	constexpr auto operator()() const
	{
		return __value*__value;
	}
};

int main()
{
	constexpr kjcpp::fx32 value = 123.456f;
	constexpr auto p2 = power<value>{}();
	kjcpp::print(p2);
}

output: 

15241.383789

example: kjcpp::class_type inheritence, polymorphism, pointer

kjcpp::class_type does not support polymorphism itself, but it can be used as the base class of another polymorphism base class. 

#include <kjcpp/core.hpp>
#include <memory>

class base: public kjcpp::ix64
{
public:
	base(const auto & value): kjcpp::ix64{value}
	{
	}
	virtual ~base()
	{
		kjcpp::print("~base");
	}
};

class derived: virtual public base
{
public:
	derived(const auto & value): base{value}
	{
	}
	~derived()
	{
		kjcpp::print("~derived");
	}
};

int main()
{
	base * ptr = new derived{123};
	kjcpp::print(*ptr);
	delete ptr;

	{
		std::unique_ptr<base> ptr = std::make_unique<derived>(32);
		kjcpp::print(*ptr);
	}
}

output: 

123
~derived
~base
32
~derived
~base
