kjcpp::param_mutex

kjcpp::param_mutex concept

The concept kjcpp::param_mutex is used to constrain and match a lockable mutex, to require a type of a mutex satisfy basic standard Mutex requirements. These types satisfy kjcpp::param_mutex concept: std::mutex, std::recursive_mutex, std::timed_mutex, std::shared_mutex, std::shared_timed_mutex, etc., and many more. kjcpp::param_mutex is not exactly the same as the description of standard Mutex requirements, but it works for kjcpp::print.

kjcpp::param_mutex is designed for kjcpp::print at first, but it can be used for other multithread programming too.

By passing a mutex object to kjcpp::print, that mutex will be locked and unlocked inside kjcpp::print, so make sure the mutex status is unlock when calling kjcpp::print with the mutex, unless it is a Recursive-Mutex, otherwise it's deadlock.

c++ example 

#include <kjcpp/core.hpp>
#include <kjcpp/thread.hpp>

int main()
{
	std::mutex mutex;
	auto f1 = kjcpp::async(
		[&mutex]
		{
			// mutex type is constrained by kjcpp::param_mutex of operator() of kjcpp::print.
			kjcpp::print(mutex, "Hell", "World!", 3.3);
		}
	);
}
