kjcpp::diff_size_cast

kjcpp::diff_size_cast is used to cast from one integral value to another target integral value. It requires the source value is in the scope of target type range. If the value is greater than the max value of target type, or the value is less than the min value of target type, the compilation will fail. 

template <kjcpp::integral_meric target_type, kjcpp::integral_meric auto source_value>
	requires (kjcpp::in_scope(source_value, kjcpp::min_v<target_type>, kjcpp::max_v<target_type>))
constexpr inline target_type diff_size_cast = kjcpp::meric_static_cast<target_type>(source_value);
  • target_type must be integral type
  • source_value must be integral value.
  • source_value must be a constant value.
  • source_value must be smaller than or equal to kjcpp::max_v of target_type.
  • source_value must be greater than or equal to kjcpp::min_v of target_type.

c++ example 

#include <kjcpp/core.hpp>

int main()
{
	constexpr kjcpp::i32 x = 123;
	constexpr auto x1 = kjcpp::diff_size_cast<kjcpp::i16, x>;

	constexpr kjcpp::i32 x2 = 128;

	// Error: Constraint not satisfied.
	// (Because kjcpp::i8_max is 127, x2 is greater than kjcpp::i8_max)
	auto x3 = kjcpp::diff_size_cast<kjcpp::i8, x2>;
}
