kjrun::run

kjrun::run

kjrun::run is a c++ method of kjcpp class kjrun. It is the main entry point of the program, which enables automatically exception catching.

How to use

Include kjcpp/kjrun.hpp 

#include <kjcpp/kjrun.hpp>

Implement kjrun::run method 

void kjrun::run()
{
	kjcpp::print("Hello, c++!");
}

The complete c++ example 

#include <kjcpp/kjrun.hpp>

void kjrun::run()
{
	kjcpp::print("Hello, c++!");
}

Compile code 

g++ hello.cpp -std=c++23 -o hello

Automatically try-throw-catch exception

The exception will be automatically thrown and caught. 

#include <kjcpp/kjrun.hpp>

void kjrun::run()
{
	kjcpp::print("Hello, c++!");
	throw std::runtime_error{"Hello, test c++ exception!"};
	kjcpp::print("Bye Bye!");
}

Output: 

Hello, c++!


[c++ std::exception]  Hello, test c++ exception!
============================================================

Enable tcrash: .enable_tcrash(true/false)

Enable tcrash true: after the exception is automatically throw-and-catch, rethrow it again to the system.

Enable tcrash false: after the exception is automatically throw-and-catch, do not rethrow.

Default: not enable. 

#include <kjcpp/kjrun.hpp>

void kjrun::run()
{
	self->enable_tcrash(true);
	throw std::runtime_error{"test error"};
}

output: 

[c++ std::exception]  test error
============================================================


terminate called after throwing an instance of 'std::runtime_error'
  what():  c++ std::exception: test error
Aborted (core dumped)

self member

kjrun::run is a c++ method of the class kjrun, kjrun::run is declared but not defined, it will be called by main, the user-code must define it. The member variable self is declared and defined in the class kjrun. The self is a std::shared_ptr object.

Return value: .set_return

The return value is set by method .set_return, if it has no exception.

If it has exception, the return value is non-zero, which can not be set by method .set_return. 

#include <kjcpp/kjrun.hpp>

void kjrun::run()
{
	self->set_return(0);
}
