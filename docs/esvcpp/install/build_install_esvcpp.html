c++ compilers and operating systems Requirements

Esvcpp can be built and installed on Linux, Windows, Unix, etc. This doc can be applied to all of those operating systems.

However, it is well tested on these c++ compilers:

  • Linux
    • gcc
    • dpcpp (Intel)
    • acpp (adaptivecpp)
    • clang
  • Windws
    • msys2-mingw64 gcc

Expected work on these c++ compilers:

  • Haiku OS
    • gcc
  • SerenityOS
    • gcc
    • clang

c++ language standard Requirements

  • Language Features: At least c++23
  • Library Features: At least c++23

Build and Install Esvcpp

To build and install esvcpp, you need a modern c++ compiler, and b2 build.

Get esvcpp source code

Get esvcpp source code.

Configure 

$ cd esvcpp
$ b2 help		# Show project help, provided by esvcpp.
$ b2 --help		# Show b2 program help, provided by b2 build.
$ b2 config --prefix=/sand		# config, this is not build.

Minimized Build Esvcpp Tests 

$ b2 -q -j7		# Start build.

The default build will choose minimized build, which can be applied to all supported compilers on all operating systems listed in the beginning of this doc.

Full Build Esvcpp Tests 

$ b2 -q -j7 all

Full build is done by target all, which is only applied to specific compiler on specific operating system, that it is useful for esvcpp library developer(s), and the esvcpp library users do not have to care it.

Note

Please note that when I said the esvcpp library users, I did not mean the library users are rabble. Just they do not need to full build esvcpp tests really, which can save lots of time and avoid unrelated verifications. The library developer(s) treat themself as library users sometimes, because they need it. In that case I wrote library for myself, then I use my library, so I am library user now.

Install Esvcpp 

$ su
# b2 install

Build Esvcpp Docs

To build esvcpp docs, you need b2 build and boost doc toolchain, including boostbook and quickbook.

Esvcpp docs are written with boost quickbook, they are lots of qbk files. 

$ cd <ESVCPP-SOURCE-ROOT>/docs
$ b2 -q -j7

The html docs will be generated to <ESVCPP-SOURCE-ROOT>/docs/html/ from qbk files .
