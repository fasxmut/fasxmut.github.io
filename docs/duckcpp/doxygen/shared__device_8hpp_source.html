16// 1.
17// dcpp::shared_device only do three things:
18//
19// (1). share device: make an internal reference counter.
20// (2). propagate methods of dcpp::DuckcppDevice to dcpp::shared_device
21// (3). exception handle
22//
23// Other behaviors are not allowed to add.
24
25// 2.
26// Reference counter is used to count how many objects are using the same shared underlying valid device,
27// not to count how many objects are hooked together.
28//
29// For example, 4 objects are hooked together to use same underlying device,
30// but the device is removed, so all of the 4 objects' reference counter is zero.
31//
32

137 // * The reference counter of current object is set to one,
