std::input_or_output_iterator

std::input_or_output_iterator has simple requirements: *i, ++i, i++, difference_type.

std::input_or_output_iterator concept forms the basis of the iterator concept taxonomy; every iterator type satisfies the input_or_output_iterator requirements.

Requirement List:

  • *i
  • std::weakly_incrementable
Note

Difference Type:

It is required from std::weakly_incrementable<I>:

(1) If I is iterator type, it requires I has difference_type, and its difference_type satisfies std::signed_integral.

(2) If I is not iterator type, it requires i1 - i2 type satisfies std::signed_integral, or I satsifes (1).

#include <iterator>
#include <utxcpp/core.hpp>

static_assert(! std::input_or_output_iterator<utx::ix64>);	// Unary operator* does not exist in utx::ix64.

class fizz_type
{
public:
	using difference_type = utx::i8; // std::weakly_increment requires it.
public:
	fizz_type & operator++(int)	// Does not require it returns "fizz_type &" or "fizz_type"
	{
		return *this;
	}
	fizz_type & operator++() // Requires it returns "fizz_type &"
	{
		return *this;
	}
	utx::ix32 operator*()
	{
		return 123;
	}

	fizz_type() = delete;	// OK. Does not require std::semiregular
	/*
	can not delete them:
		(Requires std::movable)

	fizz_type(const fizz_type &) = delete;
	fizz_type(const fizz_type &&) = delete;
	*/
};

static_assert(std::input_or_output_iterator<fizz_type>);

int main()
{
}
