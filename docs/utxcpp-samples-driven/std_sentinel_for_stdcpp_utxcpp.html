std::sentinel_for - What the hell is it?

std::sentinel_for concept: What the hell is it?

According to cppreference:

The sentinel_for concept specifies the relationship between an input_or_output_iterator type and a semiregular type whose values denote a range. The exposition-only concept __WeaklyEqualityComparableWith is described in equality_comparable. (cppreference) 

template< class S, class I >
	concept sentinel_for =
		std::semiregular<S> &&
		std::input_or_output_iterator<I> &&
		__WeaklyEqualityComparableWith<S, I>;

Two Parameters

std::sentinel_for concept has two parameters: S and I .

Constraints

  • S constraints: std::semiregular

Aka. It is just very basic object type concept, that requires basically copyable (support by default) and default initializable (default constructor).

  • I constraints: std::input_or_output_iterator

Aka. It is just very basic iterator type concept, that requires basically supporting difference_type, ++i, i++, *i .

  • S and I contraints: Weakly Equality Comparable

Oh, that's the point. It requires S and I are basically comparable.

Simply Overview - the Point :

std::sentinel_for requires a type S can be compared with an iterator I, but it does not require S has to be an iterator.

c++ example 

#include <iterator>
#include <utxcpp/core.hpp>

I: the iterator

First, let's write a simple basic iterator that minimally satisfies std::input_or_ouput_iterator. 

class fizz_iterator
{
public:
	using difference_type = utx::i64;
public:
	fizz_iterator & operator++()	// ++i
	{
		return *this;
	}
	fizz_iterator operator++(int)	// i++
	{
		return *this;
	}
	int operator*()	// *i	(minimally)
	{
		return 123;
	}
};

// std::input_or_output_iterator concept test.
static_assert(std::input_or_output_iterator<fizz_iterator>);

S: the sentinel (target type)

Then, let's write a basic type. basically copyable and default initializable are supported by default.

However, we have to let it comaprable with previous iterator above. 

class target_type
{
public:
	bool operator==(const fizz_iterator &) const	// comparable	(require)
	{
		return true;
	}
	// operator!= is auto deduced from operator== by default.
};

// std::semiregular concept test
static_assert(std::semiregular<target_type>);

// std::sentinel_for concept test
static_assert(std::sentinel_for<target_type, fizz_iterator>);

That's it!

Usage

At last, let's use it. 

template <typename begin_type, std::sentinel_for<begin_type> end_type>
class buzz_apply
{
public:
	constexpr buzz_apply(begin_type begin, end_type end)	// constexpr constructor
	{
	}
};

[[maybe_unused]] constexpr auto apply = buzz_apply{fizz_iterator{}, target_type{}};
