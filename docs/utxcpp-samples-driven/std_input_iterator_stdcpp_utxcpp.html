std::input_iterator

std::input_iterator concept is an iterator concept by adding readable concept, std::indirectly_readable, and a tag, std::input_iterator_tag to a basic iterator concept, std::input_or_output_iterator.

std::input_iterator concept is a refinement of input_or_output_iterator, adding the requirement that the referenced values can be read (via indirectly_readable) and the requirement that the iterator concept tag be present. (cppreference) 

template< class I >
concept input_iterator =
	std::input_or_output_iterator<I> &&
	std::indirectly_readable<I> &&
	requires { typename ???ITER_CONCEPT???<I>; } &&
	std::derived_from<???ITER_CONCEPT???<I>, std::input_iterator_tag>;

Concept Rules

  • rule-a1: std::input_or_output_iterator
  • rule-a2: std::indirectly_readable
  • rule-a3: std::derived_from std::input_iterator_tag

rule-a1 std::input_or_output_iterator

  • Has difference_type, and it is std::signed_integral
  • Has ++i
  • Has i++
  • Has *i
  • Implicit or user-defined copy/move constructors and operator= (This is the very basic object features).

Simply overview, std::input_or_output_iterator concept is just the very basic rules to make an iterator.

Note

Difference Type:

It is required from std::weakly_incrementable<I>:

(1) If I is iterator type, it requires I has difference_type, and its difference_type satisfies std::signed_integral.

(2) If I is not iterator type, it requires i1 - i2 type satisfies std::signed_integral, or I satsifes (1).

rule-a2 std::indirectly_readable

If it requires i is std::indirectly_readable, it requires *i is directly readable.

So operator* must give a value.

rule-a3 std::derived_from std::input_iterator_tag

That iterator is derived from std::input_iterator_tag.

c++ example 

#include <iterator>	// std::input_iterator
#include <utxcpp/core.hpp>

template <utx::real_meric type1>
class fizz_iterator //: public std::input_iterator_tag
{
public:
	using difference_type = utx::i64; // Require
	using value_type = type1;	// Not require
	using self_type = fizz_iterator<type1>;	// Not require
	value_type * ptr = nullptr;
public:
	fizz_iterator() = default;	// Require, but a class has implicit default initializer by default.
	fizz_iterator(type1 & some):	// Attach iterator to target.
		ptr{&some}
	{
	}

	// std::input_or_output_iterator concept requires copy/move constructors and operator=,
	// but a class has implicit copy/move constructors and operator= , if "=delete" is not used.

public:
	self_type & operator++()	// Require, it must return Lvalue-ref.
	{
		return *this;
	}
	self_type operator++(int)	// Require, but not require return Lvalue-ref or Rvalue.
	{
		return *this;
	}
	value_type operator*() const	// Require, to satisfy std::indirectly_readable.
	{
		return *ptr;
	}
};

static_assert(std::input_iterator<fizz_iterator<utx::fx64>>);

int main()
{
}
