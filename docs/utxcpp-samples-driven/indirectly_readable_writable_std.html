std::indirectly_readable, std::indirectly_writable

std::indirectly_readable - requires the type is an iterator type or pointer type and so on, and what it points to is readable.

std::indirectly_writable - requires the type is an iterator type or pointer type and so on, and what it points to is writable.

According to cppreference:

std::indirectly_readable - The concept indirectly_readable is modeled by types that are readable by applying operator*, such as pointers, smart pointers, and input iterators. (cppreference)

std::indirectly_writable - The concept indirectly_writable<Out, T> specifies the requirements for writing a value whose type and value category are encoded by T into an iterator Out's referenced object. (cppreference)

#include <utxcpp/core.hpp>
#include <iterator>

template <std::indirectly_readable type1>
class get
{
private:
	type1 info;
public:
	get(type1 info):
		info{info}
	{
	}
	auto operator()() const
	{
		return *info;
	}
};

template <typename type1, std::indirectly_writable<type1> type2>
class set
{
public:
	set(type2 info, type1 value)
	{
		*info = value;
	}
};

int main()
{
	std::vector<utx::ix32> v{123, 321, 132};
	utx::fx32 a = 123.321f;

	utx::print(get{v.begin()}(), get{&a}()); // 123 123.321

	set{v.begin(), 987};
	set{&a, 321.123f};
	utx::print(v[0], a); // 987 321.123
}
