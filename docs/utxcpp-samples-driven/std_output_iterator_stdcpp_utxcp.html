std::output_iterator

std::output_iterator concept is a refinement of std::input_or_output_iterator that requires value writable.

std::output_iterator concept is a refinement of input_or_output_iterator, adding the requirement that it can be used to write values of type and value category encoded by T (via indirectly_writable). equality_comparable is not required. (cppreference)

Requirement List:

  • rule-a1. std::input_or_output_iterator
  • rule-a2. std::indirectly_writable
  • rule-a3. *i++ = std::forward<T>(t) 
template< class I, class T >
concept output_iterator =
	std::input_or_output_iterator<I> &&
	std::indirectly_writable<I, T> &&
	requires(I i, T&& t)
	{
		*i++ = std::forward<T>(t); // not required to be equality-preserving 
	};

std::input_or_output_iterator requirements

First, the class type must satisfy std::input_or_output_iterator.

  • A basical std::input_or_output_iterator must have difference_type, and support ++i, i++, *i.
  • To satisfy std::movable, implicit or user-defined copy/move constructors and operator= must be valid, this example uses user-defined.

rule-a1: (std::input_or_output_iterator)

  • has difference_type, and it must be std::signed_integral
  • has i++
  • has ++i
  • has *i
  • user-defined or implicit copy/move constructors and operator=
Note

Difference Type:

It is required from std::weakly_incrementable<I>:

(1) If I is iterator type, it requires I has difference_type, and its difference_type satisfies std::signed_integral.

(2) If I is not iterator type, it requires i1 - i2 type satisfies std::signed_integral, or I satsifes (1). 

template <typename some_type>
class fizz_iterator
{
public:
	using difference_type = utx::i64;
private:
	some_type * ptr = nullptr;
public:
	fizz_iterator(some_type & some):
		ptr{&some}
	{
	}
public:
// User-defined copy and move constructors
	fizz_iterator(fizz_iterator<some_type> & itr):
		ptr{itr.ptr}
	{
	}
	fizz_iterator(fizz_iterator<some_type> && itr):
		ptr{std::move(itr).ptr}
	{
	}
public:
// User-defined copy and move operator=
	fizz_iterator<some_type> & operator=(fizz_iterator<some_type> & itr)
	{
		this->ptr = itr.ptr;
		return *this;
	}
	fizz_iterator<some_type> & operator=(fizz_iterator<some_type> && itr)
	{
		this->ptr = std::move(itr).ptr;
		return *this;
	}
public:
// Let i++, ++i work.
	fizz_iterator<some_type> & operator++()
	{
		// We do not actually execute ++i for this simple test, to avoid memory violation.
		return *this;
	}
	fizz_iterator<some_type> operator++(int)
	{
		// We do not actually execute i++ for this simple test, to avoid memory violation.
		return *this;
	}

std::output_iterator requirements

To support rule-a1, operator* is enough; however, to support rule-a2 and rule-a3, *i must be writable, so operator* must return a reference.

rule-a2: (std::indirectly_writable)

  • operator* returns a reference, to let it std::indirectly_writable

rule-a3: (*i++ = std::forward<T>(t))

  • *i++ = std::forward<T>(t);

*i is already required by rule-a1 and rule-a2, i++ is already required by rule-a1. Then rule-a3 requires *i++ accepts both Lvalue reference and Rvalue reference. 

public:
// Let *i work, and writable.
	some_type & operator*()
	{
		return *ptr;
	}
};

static_assert(std::input_or_output_iterator<fizz_iterator<utx::fx64>>);
static_assert(std::output_iterator<fizz_iterator<utx::fx64>, utx::fx64>);
