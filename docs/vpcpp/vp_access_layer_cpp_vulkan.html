vp::access_layer - cpp vulkan

vp::access_layer template class - cpp vulkan

template class vp::access_layer is a c++ template class, it has two template parameters, the first parameter is the vpcpp class that will be derived from the layer, the second parameter is the vk:: - vulkan.hpp class that vp::access_layer is derived from. vp::access_layer is used to create defined-ready vpcpp layer1 class. Constructor partial ready, assignment operator partial ready, comparators ready, operator ref ready, and va ready. va means variable address which returns object address, it might be this.

class vp::access_layer 

template <typename derived_api, typename vulkan_hpp_api_vpcpp>
class access_layer: virtual public vulkan_hpp_api_vpcpp;

Layered: chained inheritance

vpcpp layer1 classes will be derived from vp::access_layer, vp::access_layer will be derived from vulkan.hpp classes.

Constructors partial ready

vpcpp layer1 classes are derived from vp::access_layer(s), vp::access_layer(s) are derived from vulkan.hpp classes - vp::access_layer constructors will pre-accept parameter types of from vpcpp layer1 classes to vulkan.hpp classes . The parameters passed to vpcpp layer1 classes constructors will be propagated to vp::access_layer(s) .

Assignment operators partial ready (operator=)

vpcpp layer1 classes are derived from vp::access_layer(s), vp::access_layer(s) are derived from vulkan.hpp classes - vp::access_layer operator= will pre-accept parameter types of from vpcpp layer1 classes to vulkan.hpp classes . The parameters passed to vpcpp layer1 classes operator= will be propagated to vp::access_layer(s) .

comparators ready

The spaceship operator<=> and comparators operator== are ready to use for derived vpcpp layer1 classes. The comparators accept parameter types of from vpcpp classes to vulkan.hpp classes. Any classes that will be derived from vp::access_layer does not have to define its comparators any more.

operator ref ready

The objects created by vpcpp layer1 classes that are derived from vp::access_layer can be referenced by objects of itself, the layer, the vulkan.hpp classes (native and ex_native).

va ready

va means variable address - taking addresses of objects, the addresses of objects of vpcpp layer1 classes, vulkan.hpp classes can be taken from that method. va is ready to use for any classes that will be derived from vp::access_layer .

.va()

Taking address of itself - return this or dynamic_cast .

.va_layer()

Taking address of layer - return this or dynamic_cast .

.va_native()

Taking address of native vk::??? - return this or dynamic_cast .

.va_ex_native()

Taking address of ex_native - return address std::bit_cast from this .
